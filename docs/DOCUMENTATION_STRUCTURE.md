# ドキュメント構造ガイド（Phase 3）

このファイルは、`docs/` と `REFACTORING/` の責務を分離し、
今後のドキュメント配置ルールを統一するためのガイドです。

## 目的

- ドキュメントの正本（SoT）を明確にする
- 更新先の迷いを減らす
- 参照導線を単純化する

## 責務分離

### docs/

- 現在の運用・設計・仕様の正本
- 継続運用するガイド（例: 実行手順、検証手順、構成説明）
- 新規ドキュメントは原則 `docs/` に追加

### REFACTORING/

- フェーズ実行時の計画・記録・振り返りのアーカイブ
- 履歴として保持する内容（当時の計画、進捗記録、リリース記録）
- 既存内容の破壊的な書き換えは避ける

### archive/

- 旧版文書、退避物、静的出力、補助スクリプトの保管先
- 現行仕様の正本にはしない
- 入口は [archive/README.md](archive/README.md) と [archive/INVENTORY.md](archive/INVENTORY.md)

## 運用ルール

1. 新しい恒久ドキュメントは `docs/` に置く。
2. `REFACTORING/` には履歴・作業記録のみを置く。
3. `archive/` には旧版・退避物・出力物のみを置く。
4. 同じ主題を複数箇所で正本化しない。
5. どうしても複数箇所に必要な場合は、片方を正本にして他方は参照リンクのみにする。

## 更新時チェック

- その内容は「現行仕様」か「過去の作業記録」か
- 正本の置き場が明確か
- archive に現行手順を書き込んでいないか
- README のリンク導線が維持されているか
